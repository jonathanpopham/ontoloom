## Non-negotiable constraints (the product IS these constraints)

1. **`Cargo.toml` `[dependencies]` stays empty.** Everything is std-only, hand-written.
   No crates, no npm, no build step for the frontend. `web/app.js` stays vanilla JS.
2. **Airgapped**: loopback only, no outbound connections, UI embedded via `include_str!`.
3. **Backward compatible**: an existing `ontoloom-graph.json` with no schema key must
   load, edit, and export exactly as before. Schema is always optional.
4. **Soft validation only**: schema violations produce warnings in the UI, never block
   saving, editing, or exporting. The domain expert is always allowed to be "wrong."
5. Match existing code style: `Result<_, String>` errors, small functions, tests in
   `#[cfg(test)]` modules per file, human-readable error messages.
